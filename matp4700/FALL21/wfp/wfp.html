1 Introduction

The United Nations World Food Program provides humanitarian foor assistance to fight hunger worldwide. This project contains a simplified model based on data provided by the authors of a recent paper [1]. The World Food Program provides aid to seven cities in Syria. Food is shipped from 3 cities close to Syria: Amman in Jordan, Beirut in Lebanon, and Gazientep in Turkey. All the other cities are in Syria.

See the figure for a map of the relevant cities. Food can be shipped in both directions along all the arcs within Syria, as well as in one direction across the borders into Syria. The aid passes through 5 regional centers, namely Aleppo, As_Suweida, Damascus, Hama, and Homs. Food can also be purchased locally and shipped within Syria, with food available at the 5 regional centers and also at markets in 3 of the final delivery points, namely Dara, Dayr_Az_Zor, and Hassakeh. (Note that we were not provided with market data for Qamishli so we do not regard it as having a local market. Also, the name of one of the cities was spelled Daraa on the map and Dara in the data provided to us.)

The populations of the 7 cities requiring aid are

City Population
Ar_Raqqa 10000
Hassakeh 10000
Dara 20000
Dayr_Az_Zor 25000
Idleb 5000
Jubb_al-Jarrah 2000
Qamishli 5000

The food supplied provides nutrients and the aim is to meet as much of the nutritional requirements in the 7 cities as possible while staying within a budget of $28,000 per month day. There are also various other constraints that the World Food Program must meet; see the details in the later sections.

In reality, aid is supplied to Syria over many months, with the price of foodstuffs varying from month to month. We simplify to consider just a single month day. We assume shipments arrive instantaneously, which is a crude approximation to a steady state situation. We don’t consider storage costs.

2 Sets

All data are contained in the file wfp.dat, available on LMS and on the course webpage. The parameter definitions in the model file are contained in the file wfp_start.mod, also on LMS and the course webpage.

The cities are divided into 4 (overlapping) sets:

  • SOURCES: the 3 cities outside Syria
  • DEMANDPOINTS: the 7 final delivery points
  • MARKETS: the 8 cities in Syria where food can be purchased by the World Food Program
  • INTERNAL: the union of DEMANDPOINTS and MARKETS

The model file also has a set NODES, which is the union of the internal and external nodes. The set of EDGES is given in the data file, along with the cost to transport 1 metric ton along each edge.

Many different types of food are available, listed in FOODS. Only some of these can be purchased inside Syria, and those are listed in FOODS_MARKET.

Each food provides a mixture of nutrients, listed in the set NUTRIENTS. Note that foods are measured in metric tons, whereas nutritional compositions are measured in units per 100 grams of food. There are 106 grams in a metric ton.

3 Objective function

The objective function is to maximize the nutrition delivered. Each city has a Nutrient Value Score (NVS), calculated by taking a simple average of the proportion of nutritional requirement met over the 12 different nutrients. The objective is to maximize the minimum NVS over the 7 demand cities. We assume each person at location i receives exactly the same basket of food. The parameter reqrj is given by the parameter demand in the data file. As a simplifying assumption, the demand per person is the same at each location. The number of people at each demand point is given by the parameter popsize.

4 Vouchers

Three of the 7 cities requiring aid have functioning markets, namely Dara, Dayr_Az_Zor, and Hassakeh. For these cities, the World Food Program has the option to provide recipients with vouchers to let them purchase goods in the markets. Each voucher is for a fraction of a metric ton, and the price for the vouchers is 99% of the cost of the World Food Program buying food directly in the markets and distributing it themselves. This value is set as a default in the file wfp_start.mod and is named vouchercost.

Thus, a food bundle provided to inhabitants of these 3 cities may include vouchers in addition to food.

5 Variables

You will probably need several sets of variables: variables for the flow of food along edges, variables for buying food locally and at the external sources, variables for vouchers, variables to capture NVS including variables for the shortfall in each nutrient at each demand point, etc.

You may also find it useful to define other (state) variables which are linear functions of these (decision) variables.

If you want your variables to be nonnegative, you have to explicitly state they are nonnegative. For example, I have nonnegative flow variables in my model:

var flow{FOODS, EDGES} >= 0;  # units of food f shipped along edge (i,j)

You can also impose simple upper bounds on variables in their definition.

6 Constraints

You will need several sets of constraints in your model.

6.1 Flow conservation

Foods can enter the network at the 3 international cities, at the five distribution points, and at the other 3 local markets. Food leaves the network at the 7 demand points. Neither the amount supplied nor the amount consumed are fixed. You need to ensure for each node v and each food that

food purchased at node v + flow into v from its neighbors
= food consumed at node v + flow from v into its neighbors.

6.2 Obey the budget

The World Food Program has a budget of $28,000 for each month day. Expenses comprise the cost of buying food and vouchers, as well as the cost of transporting food. Unit costs for food are in the parameter purchasecost for the external cities (assumed the same for each of the 3 cities) and the parameter localcost for the supply points in Syria (varies from location to location). Unit transportation costs are given by the parameter transpcost.

6.3 Calculate the nutrients supplied to each demand point

You will need to convert the food supplied at each demand point into values for each nutrient supplied. The amount of each nutrient in each food can be found in the parameter composition. Note that you will need to account for the food being measured in metric tons, while the nutritional values are given per 100g of food. You will also need to take into account popsize when calculating the nutrients supplied per person.

6.4 Ensure some minimum threshold met for each nutrient

It is required that the World Food Program meets at least 75% of the nutritional requirements for each nutrient at each demand point. In addition, it is required that at least 1900 calories per person are supplied at each location. This minimum requirement is denoted EnergyLB in the model and data files.

6.5 Obey the capacity constraints of the local markets

The three local markets that are also demand points have limited daily supplies of some foods, given by the parameter capmarket.

6.6 Obtain at least a certain proportion in Syria

In order to try to help the local economy recover, the World Food Program requires that at least 60% of the food supplied (measured by cost) is purchased in Syria, either as direct food purchases or using vouchers. The 60% value appears in the data file as the parameter localLB. Recall that the total amount spent by the World Food Program comprises the purchase costs plus the transportation costs; for this constraint, we ignore the transportation costs.

6.7 Obtain at least a certain percentage using vouchers

To further strengthen the local economy, the World Food Program requires at least 10% of the money spent on food purchases is spent in the form of vouchers. The 10% value appears in the data file as the parameter voucherLB. Again, we ignore the transportation costs in calculating this ratio.

9 AMPL notes

  • AMPL comes with an IDE, which is fairly user friendly. I believe it is integrated into the single downloads for all 3 operating systems. However, for the Mac, the IDE that comes within the ampl_macos64.tgz appears to be corrupted, so you need to also download the separate amplide.app.tgz; once you decompress this, you can move it into the ampl_macos64 folder and overwrite the Amplide already there. You don’t need to worry about paths if you use the IDE.
  • If you are working in a Windows operating system and don’t use the IDE, you can edit your model and data files in Office, for example. Save the files as plain text files. Windows may append the suffix txt to the file names, in which case you would need to include that suffix when asking ampl to read the file.
  • If you want to read in a new model file chips.mod and data file chips.dat, you can use the reset command:

    ampl: reset;
    ampl: model chips.mod;
    ampl: data chips.dat;

  • If you want to reset the whole data file and read in a new data file chips.dat, but you want to keep the model file as before, you can type

    ampl: reset data;
    ampl: data chips.dat;
