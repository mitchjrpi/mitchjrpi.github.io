1 Introduction

One of the challenges in responding to a natural disaster is in the removal of debris from roads to allow access to people in need of help. It is critical that road clearing be performed quickly, although there may be limited trucks available immediately after the disaster. In this project, you will develop an optimization model to determine a set of roads to be cleared in order to reach people in need as efficiently as possible.

The model can be written more exactly as an integer program with binary variables. For the first part of the project, you will relax the integrality constraints, instead using simple bounds 0 ≤ xj ≤ 1 on all the variables xj.

3 The data

The data instance consists of a directed graph with 600 nodes and 1150 arcs. You can think of the nodes as arranged in a 30 × 20 grid, with all arcs pointing either down or to the right. The 49 nodes on the top and the left hand side are possible access points. There are 10 demand points scattered through the grid. Because of resource limitations, at most 4 access points can be opened; this is the number of trucks available for road clearance. Each opened access point can supply at most 10 units of goods.

Files containing the list of nodes, the list of access points, the list of demand points with their demands, the clearance and transportation times can be found on LMS, and also on the course webpage at

http://www.rpi.edu/~mitchj/matp4700/projectdata

Also available is a small version of the problem, which might be useful for debugging your model.

(An aside on the units for the demands: The stated demands are daily demands for each destination. Each open access point can send out up to 10 units of goods each day. The access points have plenty of trucks available for transportation of goods, so they can send supplies to each demand point each day.)

4 Constraints and Variables

The following constraints are necessary:

  1. Roads can only be used for transportation if they have been cleared.
  2. There is a limit on the number of access points that can be opened.
  3. The supply limit of 10 has to be observed at each open access point.
  4. The demands for each destination must be met.
  5. The demand flows for each destination point must satisfy flow conservation:

    if the demand is dk for destination k then:

    • k units must leave a supply node destined for k.
    • k units must arrive at destination k.
    • for every other node, for the goods destined for k: what comes in is equal to what goes out.

Also note that the objective function is piecewise linear, so you will need to represent this using linear constraints.

Ideally, each demand node should be supplied along a single path. We can use binary variables with three indices (k,i,j) to indicate whether the flow to destination k uses arc (i,j). Thus, I suggest using variables of the form flow(k,i,j) for each destination k and each arc (i,j).
