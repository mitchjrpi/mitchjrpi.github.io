The solution found in Part 1 has fractional values for some of the variables. In this part, we investigate methods to try to tighten up the formulation.

If you prefer to use them, my model and run file are available on the course website. The solution to the LP relaxation has value 166.04. If your model returns a different value, you need to either modify your model so it does give this value, or use my model. The output obtained by my model is available online.

My model has four sets of variables which are binary in the IP formulation and between 0 and 1 in the LP relaxation:

  • useorigin{i in ORIGINS}: indicates whether origin i is supplying relief materials. 49 variables.
  • useorigdest{k in DESTINATIONS, i in ORIGINS}: indicates whether origin i is used to supply destination k. 490 variables.
  • open{(i,j) in ARCS}: indicates whether arc (i,j) is to be cleared. 1150 variables.
  • flow{k in DESTINATIONS, (i,j) in ARCS}: indicates whether material destined for k is transported along arc (i,j). 11500 variables.

Questions

  1. Show that forcing the decisions to open the origins to be binary is not sufficient to give an integral solution, with many other variables still taking fractional values at optimality. (The variable called useorigin in my formulation is the one you can set to be binary.)
  2. Is there one set of variables that can be set to be binary which results in an integer solution?
  3. Return again to the LP relaxation, with all variables taking values between 0 and 1.
    1. Why must a feasible solution to the integer program have at least 4 open origins? What linear constraint would you add to the model to enforce this? Does this improve the value of the LP relaxation?
    2. Why must the subset {173, 265, 359, 475, 559, 591} of the destinations be supplied by at least 3 open origins?
