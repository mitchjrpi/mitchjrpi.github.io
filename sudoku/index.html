Solving Sudoku using AMPL

A sudoku problem can be modeled as an integer programming feasibility problem.

Some model files:

  1. sudoku.mod: known entries in the square are input in triples. Eg, if square(3,2) contains a 5, then the KNOWN[5,2,3]=1.
  2. sudokuInit.mod: known entries are input in a 9x9 grid, with zeroes representing the unknown entries.
  3. sudokuI2.mod: as for sudokuInit.mod, except for the objective function.
  4. sudokuI2b.mod: as for sudokuI2.mod, with the binary restriction on the variables imposed explicitly.
  5. sudokuQ.mod: as for sudoku.mod, except with a quadratic objective
  6. sudokuQI.mod: as for sudokuInit.mod, except a quadratic objective is used, to try to find alternative feasible solutions to the LP relaxation.

Data files:

  • sudoku2.dat, sudoku3.dat, and sudoku4.dat are input files for sudoku.mod. These problems are solved by the CPLEX LP preprocessor.
  • sudokuI.dat, sudokuI2.dat, and sudokuI3.dat are input files for sudokuInit.mod. For these problems, the solution to the LP relaxation is nonintegral. These problems are taken from here.
